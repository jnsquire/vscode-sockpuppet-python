    GitHub Pages Setup for Documentation¶

    This guide explains how to configure GitHub to automatically build and publish your documentation.

    Option 1: Manual Deployment (Simple)¶

    Step 1: Enable GitHub Pages¶

    1. Go to your repository on GitHub
    2. Click Settings → Pages (in the left sidebar)
    3. Under Source, select:
    4. Branch: gh-pages
    5. Folder: / (root)
    6. Click Save

    Step 2: Deploy from Local Machine¶

    cd python
python docs.py deploy

    This command: 1. Builds the documentation 2. Creates/updates the gh-pages branch 3. Pushes to GitHub 4. GitHub Pages automatically serves the content

    Step 3: Access Your Documentation¶

    After a few minutes, your docs will be available at: - https://<username>.github.io/<repository-name>/ - For example: https://jnsquire.github.io/vscode-sockpuppet-python/

    Option 2: Automatic Deployment with GitHub Actions (Recommended)¶

    This automatically rebuilds and deploys documentation whenever you push to main.

    Step 1: Create GitHub Actions Workflow¶

    Create .github/workflows/docs.yml in your repository:

    name: Build and Deploy Documentation

on:
  push:
    branches:
      - main
    paths:
      - 'python/**'
      - '.github/workflows/docs.yml'
  workflow_dispatch:  # Allow manual trigger

permissions:
  contents: write

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Full history for proper git operations

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
          cache: 'pip'

      - name: Install dependencies
        run: |
          cd python
          pip install -e ".[docs]"

      - name: Build and deploy documentation
        run: |
          cd python
          mkdocs gh-deploy --force --clean

    Step 2: Configure GitHub Pages¶

    1. Go to Settings → Pages
    2. Under Source, select:
    3. Branch: gh-pages
    4. Folder: / (root)
    5. Click Save

    Step 3: Commit and Push¶

    git add .github/workflows/docs.yml
git commit -m "Add documentation deployment workflow"
git push

    The workflow will automatically run and deploy your docs!

    Option 3: GitHub Actions with Custom Domain (Advanced)¶

    If you have a custom domain like docs.yourdomain.com:

    Step 1: Update mkdocs.yml¶

    Add your custom domain:

    site_url: https://docs.yourdomain.com

    Step 2: Create CNAME file¶

    Create python/docs_src/CNAME with your domain:

    docs.yourdomain.com

    Step 3: Configure DNS¶

    Add a CNAME record in your DNS settings: - Type: CNAME - Name: docs (or your subdomain) - Value: <username>.github.io

    Step 4: Configure GitHub Pages¶

    1. Go to Settings → Pages
    2. Enter your custom domain
    3. Check "Enforce HTTPS" (after DNS propagates)

    Workflow Features Explained¶

    Trigger Conditions¶

    on:
  push:
    branches:
      - main              # Only deploy from main branch
    paths:
      - 'python/**'       # Only when Python code changes
  workflow_dispatch:      # Allow manual trigger from GitHub UI

    Permissions¶

    permissions:
  contents: write  # Required to push to gh-pages branch

    Optimizations¶

    cache: 'pip'  # Cache pip packages for faster builds
fetch-depth: 0  # Full git history for mkdocs gh-deploy

    Monitoring Deployments¶

    View Workflow Status¶

    1. Go to Actions tab in your repository
    2. Click on the latest "Build and Deploy Documentation" run
    3. View logs to see build progress and any errors

    Check Deployment Status¶

    1. Go to Settings → Pages
    2. You'll see: "Your site is live at https://..."
    3. Click the URL to view your docs

    Troubleshooting¶

    Issue: "gh-pages branch not found"¶

    Solution: Run manual deployment once to create the branch: 

    cd python
python docs.py deploy

    Issue: "Permission denied to github-actions[bot]"¶

    Solution: Check repository settings: 1. Go to Settings → Actions → General 2. Scroll to "Workflow permissions" 3. Select "Read and write permissions" 4. Click Save

    Issue: "Documentation not updating"¶

    Solutions: 1. Check the Actions tab for errors 2. Clear your browser cache 3. Wait 1-2 minutes for GitHub Pages to rebuild 4. Verify the workflow triggered (check paths filter)

    Issue: "404 Page Not Found"¶

    Solutions: 1. Verify GitHub Pages is enabled in Settings → Pages 2. Check that gh-pages branch exists 3. Ensure the branch contains index.html 4. Wait a few minutes for deployment to complete

    Best Practices¶

    1. Test Locally First¶

    Always test documentation locally before deploying: 

    python docs.py serve

    2. Use Branch Protection¶

    Protect your gh-pages branch: 1. Go to Settings → Branches 2. Add rule for gh-pages 3. Enable "Require status checks to pass"

    3. Add Status Badge¶

    Add a badge to your README showing build status:

    [![Documentation](https://github.com/<user>/<repo>/actions/workflows/docs.yml/badge.svg)](https://github.com/<user>/<repo>/actions/workflows/docs.yml)

    4. Version Documentation¶

    For version-specific docs, use mike:

    pip install mike
mike deploy --push --update-aliases 0.1 latest
mike set-default --push latest

    Alternative: GitHub Pages from Main Branch¶

    Instead of using gh-pages branch, you can serve docs from main branch:

    Step 1: Change mkdocs output directory¶

    In mkdocs.yml: 

    site_dir: docs  # Instead of default 'site'

    Step 2: Update .gitignore¶

    Remove docs/ from .gitignore if present.

    Step 3: Build and commit¶

    python docs.py build
git add docs/
git commit -m "Build documentation"
git push

    Step 4: Configure GitHub Pages¶

    1. Go to Settings → Pages
    2. Under Source, select:
    3. Branch: main
    4. Folder: /docs

    ⚠️ Note: This approach commits built HTML to your repo, which can bloat history. The gh-pages branch method is generally preferred.

    Security Considerations¶

    1. Secrets¶

    If your docs need API keys or secrets: - Store them in GitHub Secrets (Settings → Secrets → Actions) - Access in workflow: ${{ secrets.SECRET_NAME }}

    2. Branch Protection¶

    Protect your main branches to prevent accidental doc overwrites.

    3. HTTPS¶

    Always enable "Enforce HTTPS" in GitHub Pages settings.

    Next Steps¶

    1. Choose your deployment method (manual or automated)
    2. Create the workflow file if using GitHub Actions
    3. Configure GitHub Pages in repository settings
    4. Test the deployment by pushing a change
    5. Add documentation URL to your README

    Documentation URLs¶

    After setup, update these files with your docs URL:

    pyproject.toml: 

    [project.urls]
Documentation = "https://jnsquire.github.io/vscode-sockpuppet-python/"

    README.md: 

    📚 [Documentation](https://jnsquire.github.io/vscode-sockpuppet-python/)
